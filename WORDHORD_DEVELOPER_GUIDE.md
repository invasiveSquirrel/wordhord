# 🎓 Coding Teacher's Guide to Wordhord & Vocabulary Generation

Welcome to the **Wordhord** codebase! This project is a specialized language-learning ecosystem designed to generate, organize, and practice vocabulary using state-of-the-art AI.

---

## 🏗️ Project Architecture

The system is divided into two main units that work together:
1.  **The Generation Unit**: A background factory that creates vocabulary data.
2.  **The Application Unit (Wordhord)**: A database and user interface to manage and study that data.

### 📁 Folder Structure

```text
wordhord/
├── wordhord.db             # 🗄️ THE MASTER DATABASE (SQLite)
├── wordhord_api.txt        # 🔑 Your Gemini API Key
├── start.sh                # 🚀 The "Big Green Button" to launch everything
├── generate_vocabulary_improved.py  # 🏭 The AI Data Factory
├── migrate_to_sqlite.py    # 🚚 The Data Mover (Syncs Markdown to Database)
├── cleanup_database.py     # 🧹 The Data Polisher (Fixes formatting errors)
├── backend/                # 🧠 The Server (Python/FastAPI)
│   └── main.py             # Logic for API calls, audio, and database CRUD
├── frontend/               # 🖥️ The User Interface (React/TypeScript)
│   ├── src/App.tsx         # Main UI logic and Flashcard display
│   └── src/CardEditor.tsx  # The pop-up form for editing/adding words
└── archive/                # 📦 History of old logs and scripts
```

## 🏭 Unit 1: The Generation Unit

### `generate_vocabulary_improved.py`
This is your **Data Factory**. It runs in the background and talks to the **Gemini 2.0 Flash API**.

*   **How it works individually**: 
    1. It reads your raw source texts (from `/home/chris/vocabulary_sources`).
    2. It tracks its progress by counting how many entries exist in your Markdown files (`polyglossia/*.md`).
    3. It sends a "Linguistic Extraction" prompt to Gemini. 
*   **The Algorithm**: 
    *   **Context Sliding Window**: It calculates a `start_line` based on your current word count to "slide" through the source books, ensuring it doesn't repeat the same words.
    *   **Rate-Limit Management**: It uses `time.sleep()` and `try-except` blocks to handle "429 Resource Exhausted" errors, pausing when the API gets tired.
*   **API Usage**: It uses the `google-generativeai` library. The model is configured using the key in `wordhord_api.txt`.

---

## 🚚 The Bridge: The Migration Unit

### `migrate_to_sqlite.py`
Data generated by the factory is just text in Markdown files. To use it in an app, it must be in a **Database**.

*   **The Logic**: It uses **Regular Expressions (Regex)** to parse the Markdown bullet points.
*   **The "Aggressive Cleanup" Loop**: 
    *   Since AI sometimes adds extra notes like `[A1]` or `-`, this script uses a `while True` loop to repeatedly strip unwanted characters until the word is "pure."
    *   **German Logic**: It identifies nouns (by checking the "Part of Speech" or "Gender" fields) and automatically ensures they start with a lowercase article (`der`, `die`, `das`) and a capitalized word.

## 📱 Unit 2: The Application Unit (Wordhord)

### The Backend (`backend/main.py`)
This is the "Brain" of the app. It provides endpoints (URLs) that the UI calls to get or save data.

*   **SQLAlchemy ORM**: Instead of writing raw SQL, we use Python classes (`CardModel`) to represent database tables. This makes the code easier to read and maintain.
*   **CRUD Operations**:
    *   **C**reate: `POST /cards`
    *   **R**ead: `GET /cards/{language}`
    *   **U**pdate: `PUT /cards/{id}`
    *   **D**elete: `DELETE /cards/{id}`
*   **SM-2 Algorithm (Light Version)**: When you click "Daily Review," the server calculates `next_review` using an `ease_factor` and `interval`. If you get a word right, the interval grows; if wrong, it resets to 1 day.

---

### The Frontend (`frontend/src/App.tsx`)
This is the "Face" of the app. 

*   **State Management**: Uses React `useState` to track which language you're studying, which card is active, and whether the "Editor" is open.
*   **The "Smart Reload"**: Clicking "Reload" doesn't just refresh the page. It sends a request to the backend to trigger `migrate_to_sqlite.py`, bringing in new words from the factory automatically.

## 🛠️ How to Maintain & Edit

1.  **To add a new language**:
    *   Update the `LANGUAGES` dictionary in `generate_vocabulary_improved.py`.
    *   Add the language to the `LANGUAGES` array in `App.tsx`.
2.  **To fix formatting**:
    *   Edit `cleanup_database.py` and run it. This script is your "safety net"—if the AI makes a mess, this script cleans it up across all 10,000+ words in seconds.
3.  **To change the AI's behavior**:
    *   Edit the `prompt` string in `generate_vocabulary_improved.py`. This is the "brain surgery" area where you tell the AI exactly how to format the IPA, Tones, or Gender.
